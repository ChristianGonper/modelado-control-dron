# Phase 1 Baseline: Dron Generico De Referencia

## Purpose

This document fixes the official baseline vehicle used by Phase 1 and all later phases that depend on the same physical reference. The baseline is a single scenario profile, not a parallel contract.

## Baseline Profile

- `generic-drone-baseline-v1`
- Scenario metadata name: `generic-drone-baseline-v1`
- Scenario tags: `baseline`, `generic-drone`, `phase-1`

The profile is expressed through the existing `SimulationScenario` contract and the existing rigid-body vehicle parameters.

## Physical Baseline

| Parameter | Value | Unit | Meaning | Expected effect |
|---|---:|---|---|---|
| Mass | `1.0` | `kg` | Vehicle mass used by translational dynamics | Sets hover thrust and acceleration response |
| Inertia | `(0.02, 0.02, 0.04)` | `kg m^2` | Principal rigid-body inertias about body axes | Sets rotational acceleration and coupling strength |
| Gravity | `9.81` | `m/s^2` | Local gravity used by the plant | Defines the hover thrust target |
| Arm length | `0.11` | `m` | Rotor offset from the center of mass | Sets roll/pitch authority |
| Rotor thrust coefficient | `2.5e-6` | `N / (rad/s)^2` | Thrust generated by each rotor speed | Maps actuator speed to force |
| Rotor reaction torque coefficient | `1.0e-7` | `N m / N` | Yaw reaction torque per unit thrust | Sets yaw coupling from the rotors |
| Rotor motor inertia | `1.0e-5` | `kg m^2` | Rotor spin inertia | Affects actuator lag response |
| Motor time constant | `0.03` | `s` | First-order motor lag used by the actuator model | Slows thrust changes without changing the control contract |
| Max rotor speed | `1500.0` | `rad/s` | Per-rotor actuator ceiling | Caps per-rotor thrust |
| Max collective thrust | `20.0` | `N` | Global thrust limit of the vehicle | Must stay below the total rotor capacity |
| Max body torque | `(0.3, 0.3, 0.2)` | `N m` | Global torque limits in body axes | Bounds the cascade controller output |

## Control Baseline

- The official controller baseline remains the existing cascaded controller.
- The command contract stays at high level: collective thrust plus body torques.
- Control gains can change inside the same baseline profile if the physical vehicle definition is unchanged and the campaign is still comparable.

## Experimental Configuration And Traceability

The baseline scenario records enough metadata to identify it in downstream artifacts:

- scenario name
- scenario tags
- baseline profile id in trajectory parameters
- serialized vehicle parameters and rotor geometry

### Comparable Changes

The following changes remain comparable within the same baseline campaign:

- random seed
- trajectory parameters
- controller gains
- telemetry density
- disturbance toggles that do not change the physical vehicle profile

### New Campaign Required

The following changes start a new campaign rather than a comparable variant:

- mass or inertia changes
- rotor arm geometry changes
- rotor thrust or reaction coefficients
- rotor speed ceiling changes
- motor time constant changes
- global thrust or torque limit changes
- control contract changes

## Practical Rule

If a change alters the vehicle profile, actuator envelope, or control contract, it is no longer the same baseline even if the scenario still runs.
